# WhatsApp Teachers → Google Sheets

Слушает чат родителей в WhatsApp, с помощью **Gemini** извлекает из сообщений контакты учителей (имя + телефон) и записывает их в **Google Таблицы**. Управление и просмотр — через **MCP-сервер** (Cursor и др.).

## Требования

- Python 3.11+
- [uv](https://docs.astral.sh/uv/) — менеджер пакетов и окружений

## Установка

```bash
cd /path/to/playground
uv sync
```

Скопируйте `.env.example` в `.env` и заполните переменные (см. ниже).

## Конфигурация

1. **Gemini** — [API ключ](https://aistudio.google.com/apikey) в `GEMINI_API_KEY`.
2. **Google Таблицы**:
   - Создайте [сервисный аккаунт](https://console.cloud.google.com/iam-admin/serviceaccounts), скачайте JSON и положите в `credentials.json` (или укажите путь в `GOOGLE_CREDENTIALS_PATH`).
   - Создайте таблицу, откройте доступ на редактирование для email сервисного аккаунта.
   - ID таблицы — из URL: `https://docs.google.com/spreadsheets/d/<GOOGLE_SPREADSHEET_ID>/edit`.
   - Колонки (заголовки подставит приложение, если лист пустой): **Учитель** | **Телефон/Почта** | **Роль** | **Кто написал** | **Дата сообщения** | **Источник**. «Кто написал» — имя отправителя сообщения с номером; «Дата сообщения» — когда было отправлено. Телефоны нормализуются в +972…, почта — в нижний регистр.
3. **WhatsApp Cloud API** (чтобы получать сообщения):
   - [Meta for Developers](https://developers.facebook.com/) → приложение → WhatsApp → настроить номер и webhook.
   - В `.env`: `WA_PHONE_ID`, `WA_TOKEN`, `WA_CALLBACK_URL` (публичный HTTPS, например через [ngrok](https://ngrok.com/)), `WA_VERIFY_TOKEN`. Для авто-регистрации webhook — `WA_APP_ID`, `WA_APP_SECRET`.
4. **Одна конкретная группа** (например, «הורי שכבה ה' - הדמוקרטי הפתוח»):
   - Добавь номер бизнес-аккаунта (WhatsApp Cloud API) в эту группу как участника.
   - Запусти сервер с `LOG_LEVEL=DEBUG`, отправь любое сообщение в группу — в логах появится `chat_id` (вид `120363...@g.us`).
   - Пропиши в `.env`: `WA_GROUP_ID=этот_chat_id`. Тогда обрабатываются только сообщения из этой группы.

## Несколько пользователей

Чтобы ограничить, какие чаты видит пользователь:

1. В `.env`: `AUTH_ENABLED=1`, `APP_SECRET_KEY=любая_длинная_случайная_строка`.
2. Создать пользователей: `uv run python scripts/create_user.py логин пароль` (файл `data/users.json` создаётся автоматически).
3. При необходимости отредактируйте `data/users.json` и задайте для пользователя поле `chat_ids` — список ID чатов; только они будут видны этому пользователю. Если `chat_ids` пустой или отсутствует — видны все чаты моста.

Одновременно могут быть залогинены разные пользователи (в разных браузерах/устройствах); у каждого своя сессия (cookie).

## Развёртывание на сервере

Инструкция и скрипт для деплоя по SSH (например на 159.223.0.234): **[DEPLOY.md](DEPLOY.md)**. Кратко: `./scripts/deploy.sh 159.223.0.234`, затем на сервере создать `.env`, выполнить `uv sync`, `npm install` в bridge-baileys и запустить `./run.sh` или systemd.

## Запуск

### Одна точка входа (бэкенд + подсказка про мост)

```bash
./run.sh
```

Запустит бэкенд на порту 8080; в другом терминале запусти мост (см. вывод `run.sh` или раздел «Вариант: мост» ниже).

### Webhook (приём сообщений из WhatsApp)

Сервер должен быть доступен по HTTPS. Локально — через туннель (ngrok и т.п.):

```bash
uv run uvicorn src.app:app --host 0.0.0.0 --port 8080
```

В другом терминале: `ngrok http 8080` и подставьте выданный URL в `WA_CALLBACK_URL`.

### Веб-приложение (мост + анализ чатов через Gemini)

Интерфейс: подключение моста по QR, список чатов, произвольный промпт для анализа выбранных чатов в Gemini.

1. Запусти мост (Baileys) с веб-API: в папке `bridge-baileys` выполни  
   `BACKEND_URL=http://localhost:8080 WEB_PORT=3080 node index.js`.  
   Или используй `./run.sh` — он сам поднимет мост и бэкенд.  
   Мост даёт REST API (status, chats, messages, sync, logout). При подключении Baileys получает историю через `messaging-history.set`; новые сообщения — через `messages.upsert`. Сессия хранится в `bridge-baileys/auth_baileys/`.  
   Если история приходит пустой (0 сообщений в `/api/bridge/history-stats`), отвяжи устройство в WhatsApp и запусти мост с `BROWSER_TYPE=chrome`, затем отсканируй QR заново.
2. Запусти бэкенд (если не через `run.sh`): `uv run uvicorn src.app:app --host 0.0.0.0 --port 8080`.
3. Открой в браузере: **http://localhost:8080/app/**  
   — отсканируй QR, дождись «Мост подключён», нажми «Обновить список» чатов, выбери чаты, введи промпт и нажми «Анализировать». Результат ответа Gemini отобразится на странице.  
   Если QR не появляется (например, сессия уже восстановлена), открой **http://localhost:8080/app/qr.html** или на главной нажми «Отключить мост», затем обнови страницу.

В `.env` бэкенда можно задать `BRIDGE_URL=http://localhost:3080` (значение по умолчанию), если мост слушает на другом адресе.

**Полная история через GCS:** можно один раз выгрузить все чаты в Google Cloud Storage (см. раздел «Экспорт всех чатов в GCS»). Если задан `GCS_BUCKET`, при анализе история по каждому чату сначала подгружается из GCS (полный снимок), и только при отсутствии — с моста.

**Полная история чата (с моста):** при анализе для каждого чата запрашивается до 15000 сообщений. В интерфейсе есть опция **«Перед загрузкой запросить синхронизацию с телефоном»**: мост запрашивает у телефона ещё порцию истории. Итоговая глубина ограничена тем, что **WhatsApp на связанных устройствах хранит не всю историю** — см. [справку WhatsApp](https://faq.whatsapp.com/653480766448040).

### Экспорт всех чатов в GCS

Один раз (или по расписанию) можно выгрузить **все** чаты с полной историей в Google Cloud Storage. Дальше анализ в веб-интерфейсе будет брать данные из GCS (глубина без ограничений сессии моста).

1. Создайте бакет в GCP, выдайте сервисному аккаунту (тот же, что для Таблиц — main_SA) права **Storage Object Admin** (или создание/чтение объектов) на этот бакет.
2. В `.env`: `GCS_BUCKET=имя-бакета`, при необходимости `GCS_EXPORT_PREFIX=wa-export/latest`.
3. Запустите мост и дождитесь «Мост подключён».
4. В отдельном терминале (процесс может идти часами):

```bash
uv run python -m src.export_chats_to_gcs
```

Скрипт получает список чатов с моста, для каждого запрашивает до 100 000 сообщений (с синхронизацией с телефоном), сохраняет в GCS по пути `{GCS_EXPORT_PREFIX}/{chat_id}.json`. Между чатами пауза 15 сек. Таймаут на один чат — 10 мин. После успешного экспорта при анализе в веб-интерфейсе (при включённом `GCS_BUCKET`) выбранные чаты подгружаются из GCS.

#### Как получить максимально полную доступную историю всех чатов

«Полная доступная» = всё, что WhatsApp отдаёт на связанное устройство (телефон при синхронизации подгружает историю в лимитах [WhatsApp](https://faq.whatsapp.com/653480766448040)).

**Шаги:**

1. **Перед экспортом**  
   - Телефон с WhatsApp должен быть в сети и не в режиме энергосбережения.  
   - В WhatsApp на телефоне: Настройки → Связанные устройства → ваше устройство — убедиться, что синхронизация включена (если есть опция).

2. **Запустить мост и один раз синхронизировать**  
   - Запустите мост, отсканируйте QR, дождитесь «Мост подключён».  
   - В веб-интерфейсе (http://localhost:8080/app/) выберите нужный чат, включите «Перед загрузкой запросить синхронизацию с телефоном» и нажмите «Анализировать» с любым промптом — это один раз запросит у телефона подгрузку истории для этого чата. Можно повторить для нескольких важных чатов.  
   - Либо сразу запустите экспорт в GCS (он всегда использует `sync=1` для каждого чата).

3. **Запустить экспорт в GCS**  
   ```bash
   uv run python -m src.export_chats_to_gcs
   ```  
   Для каждого чата мост вызывает `syncHistory()`, ждёт (после sync при большом лимите — дольше), затем загружает до 100 000 сообщений. В логах видно, сколько сообщений получено по каждому чату.

4. **Если нужен максимум по одному-двум чатам**  
   - После первого прогона подождите 10–30 минут (телефон может догружать историю в фоне).  
   - Запустите экспорт ещё раз: скрипт перезапишет файлы в GCS; если второй раз пришло больше сообщений — в файле будет более полная история.

5. **Ограничения**  
   - На связанных устройствах WhatsApp хранит не всю историю с телефона — очень старые сообщения могут быть только на телефоне.  
   - Один запрос на чат — до 100 000 сообщений; при таймауте скрипт повторяет попытку один раз.  
   - Для ещё большей глубины без GCS можно использовать **мост на Baileys** (`bridge-baileys`): при первом подключении WhatsApp отдаёт историю через History Sync; она хранится в памяти моста и отдаётся по тому же API. Запуск: `cd bridge-baileys && npm install && WEB_PORT=3080 node index.js` (порт 3080 — как у обычного моста, бэкенд не меняется).

### Разбор истории из JSON (офлайн)

Скрипт обрабатывает JSON-файл с историей сообщений (формат: массив сообщений с полями `body`, `from`, `from_name`, `date`). Для каждого сообщения с номером вызывается Gemini (контекст 3 до и 10 после); при признании «учитель» запись добавляется в таблицу.

Использование: сохрани историю в файл `group-history.json` в корне проекта (или укажи путь аргументом):

```bash
uv run python -m src.history_processor group-history.json
# или без аргумента — по умолчанию group-history.json
```

Для получения истории используй веб-интерфейс (выгрузка чата) или MCP-инструмент **scan_chat_history** (обрабатывает историю через мост и сразу пишет в таблицу).

### MCP-сервер (инструменты для Cursor / клиентов MCP)

Через stdio (например, для Cursor):

```bash
uv run python -m src.mcp_server
```

В Cursor добавьте в настройки MCP (например, `~/.cursor/mcp.json` или проект) сервер:

```json
{
  "mcpServers": {
    "whatsapp-teachers": {
      "command": "uv",
      "args": ["run", "python", "-m", "src.mcp_server"],
      "cwd": "/Users/kirilldemidov/playground"
    }
  }
}
```

После этого станут доступны инструменты:

- **list_teachers** — список всех записей из таблицы.
- **add_teacher** — вручную добавить учителя (имя, телефон, опционально источник).
- **parse_message_for_teachers** — отправить текст сообщения в Gemini и получить извлечённые контакты без записи в таблицу.
- **scan_chat_history** — загрузить историю группы через мост Baileys и отправить сообщения в бэкенд (Gemini + таблица). Мост и бэкенд должны быть запущены.
- **list_whatsapp_chats** — список чатов/групп с их ID (через мост Baileys).

### Мост (свой WhatsApp без Cloud API)

Если не настраивать Meta Cloud API, подключи **свой** аккаунт через мост Baileys:

1. Запусти `./run.sh` (поднимет мост и бэкенд) или вручную: бэкенд на 8080, в `bridge-baileys`: `npm install && BACKEND_URL=http://localhost:8080 WEB_PORT=3080 node index.js`.
2. Открой http://localhost:8080/app/ и отсканируй QR в WhatsApp (Связанные устройства).
3. Мост пересылает сообщения на `POST /webhook/bridge`; бэкенд разбирает их через Gemini и пишет в таблицу.

Фильтр по одной группе: `GROUP_ID` у моста или `WA_GROUP_ID` в `.env`. Обработка уже отправленных сообщений: MCP **scan_chat_history** или веб-интерфейс (выбрать чат и «Анализировать»).

## Как это работает

1. В чат (группу) родителей добавлен номер, подключённый к WhatsApp Cloud API.
2. При появлении сообщения WhatsApp шлёт webhook на ваш сервер.
3. Текст сообщения передаётся в Gemini; модель возвращает структурированный список «учитель + телефон».
4. Каждая пара записывается в Google Таблицу с проверкой дубликатов по номеру телефона.

Дедупликация — по последним 10 цифрам номера; повторные записи с тем же номером не добавляются.
